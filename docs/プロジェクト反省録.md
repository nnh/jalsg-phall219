# JALSG-PhALL219 プロジェクト反省録

**運用方針**: 本ファイルは PhALL219 試験固有の反省・改善メモを記録する。汎用化可能な知見は `docs/SAS_ClinicalTrial_ClaudeCode_共通ナレッジ.md` に転記する。

## 計画段階（〜2026-05-08）

### 良かった点

| 項目 | 内容 |
|---|---|
| EFS仕様の事前確定 | データ固定前に `TMF/spec/efs_plan_v0.4.md` でEFSロジックを確定。90例の予測値を `efs_subject_check_v0.1.csv` に整理し、CSVtoSASDS実装後のQC基準とした |
| aCRF-SDTMマッピングの早期作成 | `efficacy_mapping_v0.1.md` で各CRFフォームと SDTM の対応を事前整理。SAPレビュー時に迷いがなかった |
| SAP事前レビュー | データ固定前に16件の重大指摘＋10件のタイポを `sap_review_v0.2.md` に整理。修正前/修正後形式 + SDTM変数参照で実装プログラマが迷わないようにした |
| ADaM並存設計 | GML219ではWide-master単体だったが、PhALL219では ADaM 10種を最終出力として残す方針に。CDISC準拠＆規制対応の余地を確保 |
| 疾患別ロジック差の明示化 | AML（OVRLRESP+CR）と ALL（HEMARESP+CHR/CHRp/LFS階層）の違いを `efs_plan_v0.4` で明文化。GML219の共通ナレッジを流用しないよう注意喚起 |

### 注意点・申し送り

| 項目 | 内容 |
|---|---|
| RFS起算日の解釈 | プロトコル8.1.11.3「血液学的完全寛解(CR)到達後」をLFS到達日として運用。SAP 4.4.3もこの定義に書換が必要（2026-05-06ユーザー確定） |
| LFSのみ達成例の取扱い | RFS対象母集団に LFSのみ達成例を含めるかは SAP固定時に明文化（efs_open_issues_v0.1.md 参照） |
| dose intensity計算分母 | プロトコル9章の予定投与量を数表化したマクロ `program/macro/planned_dose.sas` が未実装 |
| FA vs AE ドメインの混同注意 | 治療相別AE Grade は FA.csv（FASPID='ae{N}'）、SAEは AE.csv（AESER='Y'）。混同するとAE集計を取り違える（共通ナレッジ §11参照） |
